Extract Metadata

POST /v1/extract

Main endpoint to scrape and parse a URL.

curl -X POST "https://metadata-foresight.p.rapidapi.com/v1/extract" \
     -H "Content-Type: application/json" \
     -H "X-RapidAPI-Key: YOUR_RAPIDAPI_KEY" \
     -H "X-RapidAPI-Host: metadata-foresight.p.rapidapi.com" \
     -d '{
           "url": "https://www.github.com",
           "enable_javascript": false
         }'
import requests

url = "https://metadata-foresight.p.rapidapi.com/v1/extract"
payload = {
    "url": "https://www.github.com",
    "enable_javascript": True
}
headers = {
    "Content-Type": "application/json",
    "X-RapidAPI-Key": "YOUR_RAPIDAPI_KEY",
    "X-RapidAPI-Host": "metadata-foresight.p.rapidapi.com"
}

response = requests.post(url, json=payload, headers=headers)
print(response.json())
const response = await fetch("https://metadata-foresight.p.rapidapi.com/v1/extract", {
    method: "POST",
    headers: {
        "Content-Type": "application/json",
        "X-RapidAPI-Key": "YOUR_RAPIDAPI_KEY",
        "X-RapidAPI-Host": "metadata-foresight.p.rapidapi.com"
    },
    body: JSON.stringify({
        url: "https://www.github.com",
        enable_javascript: true
    })
});

const data = await response.json();
console.log(data);

Parameters

Parameter Type Default Description
url string - The target URL to scrape (Required).
force_refresh bool false Skip the cache and fetch fresh data.
enable_javascript bool false Use a headless browser (Playwright) to render the page. Useful for SPAs.

Response Structure

The API returns a JSON object split into meta (request info) and data (extracted content).

{
  "meta": {
    "url": "https://github.com",
    "domain": "github.com",
    "latency_ms": 245.5
  },
  "data": {
    "title": "GitHub: Let's build from here",
    "description": "GitHub is where over 100 million developers shape the future of software...",
    "image": "https://github.githubassets.com/images/modules/site/social-cards/github-social.png",
    "favicon": "https://github.com/favicon.ico",
    "site_name": "GitHub",
    "oembed": { ... },
    "json_ld": [ ... ]
  }
}
Field Description
meta.latency_ms Time taken to fetch and parse (in milliseconds).
data.title Best available title (OG > Twitter > HTML Title).
data.image Absolute URL to the preview image.
data.oembed Rich media data (YouTube video ID, width, height) if available.
data.json_ld Structured data (e.g., Recipe, Product, Event) if found.

Guides & Use Cases

1. Building a Link Preview Card

Use the API to generate social media style cards for your chat app or forum.

// Example React Component Logic
const { data } = await api.extract(url);
return (
  <div class="card">
    <img src={data.image} />
    <h3>{data.title}</h3>
    <p>{data.description}</p>
    <span>{data.domain}</span>
  </div>
);

2. Handling Single Page Apps (SPAs)

Standard scrapers fail on sites like Instagram, TikTok, or dynamic React apps because they return empty HTML. Metadata Foresight solves this with the enable_javascript flag.

‚ö†Ô∏è Note: Headless requests take longer (2-5s) and may cost more credits on paid plans.

3. Error Handling

The API returns standard HTTP codes to help you debug.

Code Meaning
200 Success. Metadata extracted.
400 Invalid URL or Bad Request.
429 Rate limit exceeded. Slow down!
504 Timeout. The target site is too slow.
